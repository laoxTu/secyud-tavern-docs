# App Layer Design

## Overview

`src/app/` is the page and API route layer of the Next.js App Router, hosting frontend UI pages and backend RESTful APIs.

## Page Routes

```
/                          → "use client" page, immediately calls router.replace("/business")
/business                  → Main management page, initializes client plugins via PluginLayout
/business/stories/[id]     → Story conversation page, loads a Slot and runs the conversation pipeline
```

### Root Layout (`layout.tsx`)

Wraps `NextIntlClientProvider` (i18n) + `TooltipProvider` (shadcn/ui) + `Toaster` (sonner).

### Business Layout (`business/layout.tsx`)

`<PluginLayout>` — calls `useClientPlugins()` to initialize all client-side plugin systems and engines. Displays a loading iframe until initialization completes.

### Business Page (`business/page.tsx`)

Renders `<BusinessPageContent>` — iterates over `businessNavigationManager.getSorted()` to render top navigation tabs. The active tab is highlighted with `bg-gray-100`. Includes a language switcher dropdown (sets the locale cookie then reloads) and copyright info.

### Story Page (`business/stories/[id]/page.tsx`)

Renders `<StoryPageContent>` — fetches slot data → runs `conversationManager.initializer.use()` → renders the conversation iframe + bottom toolbar (history pager, output pager, slot function buttons) + chat input.

## API Route Design

### Template Factory (`api/template.ts`)

All CRUD operations are generated by factory functions, eliminating boilerplate. Each factory receives a `TemplateConfig<TModel>`:

```typescript
interface TemplateConfig<TModel> {
    repository: Repository<TModel>;
    conditionSearch?: (search) => ConditionFunc;
    conditionMatchId?: (id) => ConditionFunc;
    checkCreate?, checkUpdate?: (model) => Promise<void>;
    filename?: (model) => string;
    exportHandler?, importHandler?: ...;
}
```

**Factory Function List**:

| Function | HTTP Method | Purpose |
|---|---|---|
| `apiGetModelList` | GET | Paginated list (PageOptions from searchParams) |
| `apiGetModel` | GET /[id] | Single record details (optional withDetails) |
| `apiCreateModel` | POST | Create (auto-generates UUID, optional checkCreate validation) |
| `apiUpdateModel` | PUT /[id] | Partial update (mergeObjects for content merge) |
| `apiDeleteModel` | DELETE /[id] | Delete |
| `apiExportModel` | GET /[id]/export | File download (ReadableStream + Content-Disposition) |
| `apiImportModel` | POST /import | File upload upsert (dedup check → delete old → create new) |
| `apiGetEntryList` | GET /[id]/entries/[type] | Paginated list of sub-entities |
| `apiCreateEntry` | POST /[id]/entries/[type] | Create sub-entity (auto-increment entryId) |
| `apiUpdateEntry` | PUT /.../[entryId] | Update sub-entity |
| `apiDeleteEntry` | DELETE /.../[entryId] | Delete sub-entity |
| `apiDisableEntry` | PUT /.../disabled | Toggle disabled state |

### Middleware Wrapping

All API routes are wrapped with `interceptor.createRoute(handler)`, which automatically applies:

1. **ErrorInterceptor** — catches BusinessError → JSON `{message, code, data}`
2. **ParamInterceptor** — parses `nextUrl.searchParams`, attempts JSON.parse on each value

```typescript
// Typical route file
export const GET = interceptor.createRoute(apiGetModelList(apiConfig));
```

### Domain APIs

**Stories** (`/api/stories/`):
- Standard CRUD + import/export + sub-entity management (history entries)
- `/api/stories/[id]/slot` — **Slot assembly endpoint**: fetches story → recursively resolves all presets by requires via BFS → fetches llmapi → loads history entries → returns the complete SlotModel
- Validation: name must be non-empty
- Search: name LIKE fuzzy match

**Presets** (`/api/presets/`):
- Standard CRUD + import/export + sub-entity management (lorebooks/regexes/styles/scripts/macros entries)
- **PNG export**: if a cover image exists, streams image bytes first, then appends JSON data bytes
- **PNG import**: uses `splitPNGAndDataUniversal` to separate image and JSON; image is saved to imageRepository
- Validation: code and name must be non-empty, code uniqueness check
- Search: name + code fuzzy match, tags array overlap filter
- ID matching: supports UUID or readable code

**LLM APIs** (`/api/llmapis/`):
- Standard CRUD + import/export + sub-entity management
- `/api/llmapis/[id]/chat` — **Chat endpoint**: looks up config → finds engine in `llmapiEngineRegistry` by provider → `Hasher.instance.decrypt(key)` to decrypt → `engine.run(messages, config, apiKey, signal)` → returns `text/event-stream` SSE stream (Cache-Control: no-cache, Connection: keep-alive)
- Validation: code and name must be non-empty, code uniqueness check
- ID matching: supports UUID or readable code

**Images** (`/api/images/`):
- POST — reads ArrayBuffer, determines MIME from Content-Type, `imageRepository.create()`, SHA-256 dedup
- GET /[id] — returns image, 1 year cache (max-age=31536000, immutable)
- DELETE /[id] — deletes file and record

**Plugins** (`/api/plugins/`):
- GET — returns `pluginManager.getPlugins()` JSON list
- GET /[pluginId] — reads the clientScript file from the plugin directory, returns `application/javascript` (Cache-Control: no-cache, no-store, must-revalidate)
